An analogy to describe the differences between HTML and CSS

If a website is a house...

  • HTML is the framework of your house - it sets the structure of your house - what rooms there are, what is in each of the rooms - the general layout of the place
  • CSS is the decorator of your house - decides the colours and sizes of elements, how much free space there is, the positioning of elements within the room

Explain control flow and loops using an example process from everyday life

Control flow is the order in which pieces of code are executed

Loops allow us to execute a certain piece of code multiple times, based on conditions. We could execute the code a set number of times, or execute the code until a condition is met. We can also have several pieces of code in one loop - which piece of code gets executed depends on which conditions are met.

We could think about control flow and loops in terms of making a coffee in the morning

  1. Go to kitchen
  2. Check jug status - is there enough water?
  3. If yes, turn on jug. If no, fill up jug with enough water, then turn on jug.
  4. Check plunger status - is it available? Is it clean?
  5. If not available, buy a new one. If not clean, clean it.
  6. Then fill with appropriate amount of coffee.
  7. Check jug status - has it boiled?
  8. If no, wait until boiled. If yes, fill plunger with hot water.
  9. Wait 3 minutes.
  10. Push plunger down, pour coffee into mug.
  11. Insert coffee in mouth.

Describe what the DOM is and an example of how you might interact with it

The Document Object Model treats an HTML file as a tree structure, where each element on the page is a node on the tree. The DOM has a hierarchy - has parents, children and ancestors. Eg the body of an HTML document has the HTML document as its parent, and elements like headings, paragraphs, images etc as its children.

We could use HTML, the DOM and JavaScript to create a button (HTML), that when pressed changes all the paragraphs in the document (DOM) to hotPink (JavaScript)

Explain the difference between accessing data from arrays and objects

Arrays and objects are both mutable - ie they can be changed easily - and can both be used to store a list of values.

Objects are best used to represent a 'thing' - like a person, car, or other 'object' in the world. A 'thing'/object has a set of characteristics that are called properties - in the example of a person these could be their name, age, height, weight, eye colour, favourite Spice Girl etc eg:
person = {
name: 'Casey',
lives: 'Wellington',
spiceGirl: 'Sporty'
}

Data in an object can be accessed by dot or bracket notation

eg person.name would return 'Casey'

person[lives] would return 'Wellington'

Arrays are lists of multiple items in a single variable eg newZealandCities = ['Auckland', 'Hamilton', 'Wellington', 'Christchurch', 'Dunedin', 'Gore']

Data in arrays can be accessed by its index value in square brackets - remembering that computers start counting at 0 - so newZealandCities[2] would return 'Wellington'

Explain what functions are and why they are useful

Functions are little programs within websites that do small tasks for us

They let us do cool things with websites - they add interactivity, because they can 'remember' things, and use the computers hardware to 'think', or process information. So instead of a webpage being 'static' with just HTML, or static but pretty with CSS (and slightly interactive), with functions in JavaScript we can create websites that have interactivity and can store and process information.

eg a supermarkets website:

  • with just HTML we could view the products
  • with HTML and CSS we could view the products on a pretty and responsive site
  • with HTML, CSS and JavaScript we could add them to our shopping cart and give them shipping and payment details so they can deliver the products to our house.
