CSS Positioning

The position property in CSS specifies the positioning method used for an element.

There are 6 position values:

  • static
  • relative
  • absolute
  • fixed
  • sticky
  • inherit

Today we're going to look at relative, absolute and fixed positioning.

Positioning allows you to take elements out of the normal layout flow of a document, and make them behave differently. You could have an element sitting on top of another one, or have an element always remain in the same position on a users screen.

You set which position method using the position: property, followed by which method you'd like to use

You set the position using top, right, bottom and left properties. It is important to note that your element will move AWAY from the property by the specified amount eg top: 30px; will push the element DOWN by 30px.

Positioning doesn't cascade, ie child elements will not inherit a position from their parent, unless you tell the child specifically to inherit it.

Relative Positioning

What is it?

CSS elements with relative positioning are positioned relative to where they would normally be placed, ie 'relative to itself'. Other content will not be adjusted to fit into any gap left by the element moving.

What's it good for?

  • Offsetting elements
  • Getting elements to line up

How can I use it?

  • position: relative; sets the element to have a relative position
  • top, right, bottom and left properties are used to set the elements position, relative to where they would normally be placed

If you set the element to a relative position, and don't set any other position attributes (top, right, bottom or left), nothing will happen - the element will stay in its 'normal' position.

Absolute Positioning

What is it?

CSS elements with absolute positioning are positioned relative to the nearest relatively (or absolutely) positioned parent. If it has no positioned parents, then it uses the document body.

What's it good for?

This powerful type of positioning allows you to place your element precisely where you want it.

The trade off it the element is removed from the flow of elements on the page. An element with an absolute position is not affected by other elements, and doesn't affect other elements. The element will overlap other elements unless you take action to avoid it.

Overuse can limit the flexibility of your site.

It can be used for popup info boxes, control menus, rollover panels etc.

How can I use it?

  • position: absolute; sets the element to have absolute positioning
  • top, right, bottom and left properties are used to set the elements position, relative to its parent.

Fixed Positioning

What is it?

CSS elements with fixed positioning are similar to absolute positioning - instead of being relative to their parent, they are fixed relative to the users viewport, or screen. This means that the element stays in the same position on the users screen, even when scrolling. It doesn't leave a gap in the page where it would have been located.

What's it good for?

  • A navbar you want to always remain at the top of the page, even when scrolling
  • A footer that is always visible, and might have copyright info, contact links, social media links etc

How can I use it?

  • position: fixed; sets the element to have a fixed position
  • top, right, bottom and left properties are used to set the elements position, relative to the users viewport/screen.
